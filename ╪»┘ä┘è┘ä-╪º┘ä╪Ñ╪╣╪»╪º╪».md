## 1️⃣ تثبيت Node.js

Node.js هو البرنامج اللي بيشغّل التطبيق.

1. افتح الموقع: **https://nodejs.org**
2. اضغط على زر **LTS** (النسخة الموصى بها)
3. شغّل ملف التثبيت بعد التحميل
4. اضغط Next على كل الخطوات (الإعدادات الافتراضية مظبوطة)
5. بعد التثبيت، افتح **Terminal** (على Mac) أو **Command Prompt** (على Windows)
6. اكتب الأمر التالي للتأكد:
   ```
   node --version
   ```
   لازم يظهر رقم النسخة (مثلاً `v20.10.0`). لو ظهر، تمام.

## 2️⃣ إنشاء مشروع Firebase

Firebase هو قاعدة البيانات اللي هتحفظ فيها بيانات المهام مجاناً.

### الخطوة 2.1: إنشاء المشروع

1. افتح **https://console.firebase.google.com**
2. سجّل الدخول بحساب جوجل
3. اضغط **"Add project"** (أو "إنشاء مشروع")
4. اكتب اسم للمشروع، مثلاً: `daily-tasks-tracker`
5. اضغط **Continue**
6. **عطّل** Google Analytics (مش محتاجينه) - اضغط على التوغل عشان يبقى Off
7. اضغط **Create project**
8. استنى 20-30 ثانية، هيظهر "Your new project is ready"
9. اضغط **Continue**

### الخطوة 2.2: تفعيل Firestore Database

1. من القائمة الجانبية على الشمال، اضغط على **Build** ثم **Firestore Database**
2. اضغط **Create database**
3. **هام:** اختر **"Start in test mode"** (مش production mode)
4. اضغط **Next**
5. اختر موقع السيرفر الأقرب لك - مثلاً `eur3 (europe-west)` للشرق الأوسط
6. اضغط **Enable**
7. استنى 30 ثانية، Firestore هيتفعّل

### الخطوة 2.3: ضبط Security Rules (مهم جداً!)

التطبيق محتاج صلاحيات يكتب ويقرأ من قاعدة البيانات.

1. من نفس صفحة Firestore، اضغط تبويب **Rules** فوق
2. امسح المحتوى الموجود واستبدله بالكامل بهذا:
   ```
   rules_version = '2';
   service cloud.firestore {
     match /databases/{database}/documents {
       match /{document=**} {
         allow read, write: if true;
       }
     }
   }
   ```
3. اضغط **Publish**

> **ملاحظة أمنية:** هذه القواعد تسمح لأي حد عنده اللينك بالقراءة والكتابة. ده مناسب لمجموعة صغيرة موثوقة. لو محتاج أمان أعلى لاحقاً، نقدر نضيف Firebase Auth.

### الخطوة 2.4: الحصول على مفاتيح الاتصال

1. اضغط على أيقونة الترس ⚙️ بجانب "Project Overview" → **Project settings**
2. انزل لتحت لقسم **"Your apps"**
3. اضغط على أيقونة الويب **`</>`**
4. اكتب اسم للتطبيق، مثلاً: `tasks-web`
5. **لا تحدد** "Also set up Firebase Hosting"
6. اضغط **Register app**
7. ستظهر شاشة فيها كود يشبه هذا:
   ```javascript
   const firebaseConfig = {
     apiKey: "AIzaSyD...",
     authDomain: "your-project.firebaseapp.com",
     projectId: "your-project",
     storageBucket: "your-project.appspot.com",
     messagingSenderId: "123456789",
     appId: "1:123:web:abc"
   };
   ```
8. **انسخ كل القيم** - هتحتاجها في الخطوة التالية
9. اضغط **Continue to console**

## 3️⃣ ضبط مفاتيح التطبيق

1. افتح فولدر التطبيق على اللاب توب
2. افتح ملف `src/firebase.js` بأي محرر نصوص (VS Code أو Notepad)
3. ستجد الكود التالي:
   ```javascript
   const firebaseConfig = {
     apiKey: "ضع-المفتاح-هنا",
     authDomain: "ضع-القيمة-هنا.firebaseapp.com",
     ...
   };
   ```
4. **استبدل كل القيم** بالقيم اللي نسختها من Firebase في الخطوة السابقة
5. احفظ الملف (Ctrl+S)

### 🔐 غيّر كلمة مرور المشرف

1. افتح ملف `src/App.jsx`
2. ابحث عن السطر:
   ```javascript
   const ADMIN_PASSWORD = "admin123";
   ```
3. غيّر `admin123` لكلمة سر قوية تختارها (مثلاً `Pink2026!Tasks`)
4. احفظ الملف

> 💡 هذه كلمة المرور التي ستستخدمها المشرفة للدخول إلى لوحة المتابعة والإعدادات. الأفراد العاديون **لا يحتاجون** كلمة مرور لتسجيل مهامهم.

## 4️⃣ اختبار التطبيق محلياً

1. افتح Terminal/Command Prompt
2. انتقل لمجلد التطبيق:
   ```
   cd path/to/app-folder
   ```
3. نصّب المكتبات (مرة واحدة فقط):
   ```
   npm install
   ```
   هياخد 2-3 دقائق
4. شغّل التطبيق:
   ```
   npm run dev
   ```
5. هيظهر سطر فيه عنوان شبيه بـ `http://localhost:5173`
6. افتح هذا العنوان في المتصفح
7. لو ظهر التطبيق وشغّال، تمام جداً ✅
8. جرّب اضغط على فرد وعلّم بعض المهام للتأكد إن البيانات بتتحفظ في Firebase

> لإيقاف التطبيق المحلي: اضغط `Ctrl+C` في Terminal

## 5️⃣ رفع التطبيق على Vercel

Vercel هي الاستضافة المجانية اللي هتدّيك لينك حقيقي على إنترنت.

### الخطوة 5.1: رفع الكود على GitHub أولاً

1. افتح **https://github.com** وسجّل حساب جديد (مجاني)
2. بعد التسجيل، اضغط **+** فوق على اليمين → **New repository**
3. اكتب اسم: `daily-tasks-tracker`
4. اختاره **Private** (عشان مفاتيح Firebase لا تكون مكشوفة)
5. اضغط **Create repository**
6. ستظهر صفحة فيها أوامر - **لا تنفّذها كلها**، نفذ الأوامر التالية فقط في Terminal من داخل مجلد التطبيق:
   ```bash
   git init
   git add .
   git commit -m "first commit"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/daily-tasks-tracker.git
   git push -u origin main
   ```
   استبدل `YOUR_USERNAME` باسم حسابك على GitHub.
7. لو طلب منك تسجيل دخول، تابع التعليمات على الشاشة.

### الخطوة 5.2: ربط Vercel بـ GitHub

1. افتح **https://vercel.com**
2. اضغط **Sign Up** واختر **Continue with GitHub**
3. وافق على الصلاحيات
4. بعد التسجيل، اضغط **Add New...** → **Project**
5. ستظهر قائمة بمشاريعك على GitHub. اختر `daily-tasks-tracker` واضغط **Import**
6. لا تغيّر أي إعدادات (Vercel يكتشف Vite تلقائياً)
7. اضغط **Deploy**
8. استنى دقيقة أو دقيقتين... ✨
9. لما يخلّص، ستظهر صورة مصغّرة للتطبيق ورسالة **"Your project has been deployed"**

## 6️⃣ الحصول على اللينك والمشاركة

1. من صفحة المشروع على Vercel، انسخ اللينك (هيكون شبيه بـ `daily-tasks-tracker-abc123.vercel.app`)
2. لو حابب تغيّر اللينك لاسم أقصر:
   - اضغط على **Settings** → **Domains**
   - أضف دومين subdomain مخصص (مجاني)، مثلاً `our-tasks.vercel.app`
3. **ابعت اللينك على مجموعة الواتساب** ✅

## ❓ مشاكل شائعة

### "Permission denied" أثناء git push
- تأكد إنك سجّلت دخول GitHub في Terminal، أو استخدم GitHub Desktop كبديل أسهل

### التطبيق فتح بس البيانات مش بتتحفظ
- راجع Firestore Rules (الخطوة 2.3)
- راجع مفاتيح Firebase في `src/firebase.js`
- افتح Console المتصفح (F12) وشوف لو فيه error باللون الأحمر

### نسيت كلمة المرور
- ارجع لـ `src/App.jsx` وغيّرها واعمل push جديد
